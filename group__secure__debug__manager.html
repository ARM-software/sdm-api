Detailed Description

Top-level API for secure debug library.

The Secure Debug Manager API (SDM API) defines the interface used by debug clients, such as debuggers, IDEs, and other tools, for performing protocol-independant secure debug authentication.

◆ SDM_Open()

SDMReturnCode SDM_Open ( SDMHandle *  handle,
const SDMOpenParameters *  params 
)

This function is called by the debugger to start a secure debug session with the remote platform.

The caller is expected to set the resetType to a value other than link SDM_none when it knows that the debugged platform implements the secure debug certificate processing in its ROM. In the case of SDM_nSRSTReset, if the External COM port driver implementation does not have nSRST capability the caller can provide callbacks pDebugIf->callbacks->f_nSRSTStage1 and pDebugIf->callbacks->f_nSRSTStage2.

Parameters
[out]handleNew handle to the SDM instance.
[in]paramsConnection details and callbacks. This pointer and all pointers within the structure must remain valid until SDM_Close() is called.

◆ SDM_ResumeBoot()

SDMReturnCode SDM_ResumeBoot ( SDMHandle  handle)

Called by the debugger to resume the boot of the remote platform.

It is typically called after the debugger places its breakpoints at the booting debugged system. It is only useful if the debugged system supports the introduction of debug certificate in the early boot stages, otherwise if the debugged system processes the secure debug certificate at runtime, it does not wait for the resume command.

The debugger can know this fact at the response of introduction of the secure debug certificate, which is also the return value of SDM_Open. In case it happened at early boot then SDM_Open will return SDM_SUCCESS_WAIT_RESUME. Otherwise if introduced at run time then SDM_Open will return SDM_SUCCESS.

In the 1st case, after the return of SDM_Open the user set breakpoints while the debugged system is still waiting. After breakpoints are set, the debugger can call SDM_ResumeBoot

Parameters
[in]handleHandle to the SDM instance.
